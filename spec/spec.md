# pr-watcher

Background daemon that polls GitHub for PRs needing review, drafts reviews via headless Claude Code, and waits for human approval before submitting.

## Goal

Eliminate the latency between "PR opened" and "I sit down to review it." When a PR I didn't author is ready, the daemon notifies, drafts a review, and parks it for me to approve, reject, or edit.

## Runtime target

- WSL2 on a Windows host
- Python 3.12+
- `uv`, `gh` CLI, and `claude` CLI installed and authenticated on the host
- PowerShell available via `pwsh.exe` from inside WSL
- `BurntToast` PowerShell module installed on the Windows host

## Distribution

Single-file `uv` script with PEP 723 inline metadata:

```python
# /// script
# requires-python = ">=3.12"
# dependencies = [
#   "click",
# ]
# ///
```

Run with `uv run pr-watcher.py <subcommand>`. User will symlink to `~/.local/bin/pr-watcher`.

**No `pyproject.toml`, no `setup.py`, no src layout, no package directory. One file.**

Tests live in a sibling file `test_pr_watcher.py`. Run with `uv run --with pytest pytest test_pr_watcher.py`.

## Subcommands

| Command | Behavior |
|---|---|
| `pr-watcher start` | Run daemon loop. Polls forever. |
| `pr-watcher start --once` | Single poll iteration, then exit. Used for testing and cron-style use. |
| `pr-watcher list` | Print pending reviews as JSON. |
| `pr-watcher show <pr>` | Print the draft review for a PR. |
| `pr-watcher approve <pr>` | Submit draft as approval via `gh pr review --approve`. Clean up worktree. |
| `pr-watcher comment <pr>` | Submit draft as comment-only via `gh pr review --comment`. Clean up worktree. |
| `pr-watcher request-changes <pr>` | Submit draft as request-changes via `gh pr review --request-changes`. Clean up worktree. |
| `pr-watcher reject <pr>` | Discard draft, mark PR as reviewed, clean up worktree. No GitHub submission. |
| `pr-watcher config` | Print resolved config to stdout. |

PR argument format accepts both `owner/repo#123` and `123` (uses default repo from current dir if available).

## Config

Location: `~/.config/pr-watcher/config.toml`

```toml
# Repos to watch. Format: "owner/name"
repos = [
  "dell/bpa",
  "dell/ai-bp-toolkit",
]

# Polling interval in seconds
poll_interval = 90

# Absolute path to sound files (Windows-side paths)
sound_new_pr = "C:\\Users\\adam\\sounds\\new-pr.wav"
sound_review_ready = "C:\\Users\\adam\\sounds\\review-ready.wav"

# Where to create git worktrees
worktree_base = "/home/adam/pr-reviews"

# GitHub username (PRs authored by this user are skipped)
github_user = "adam-XXX"

# Path to the review prompt template (see Review Pipeline section)
review_prompt_path = "~/.config/pr-watcher/review-prompt.md"

# Claude Code model for review
claude_model = "claude-opus-4-7"
```

On first run, if config doesn't exist, write a template with placeholders and exit with instructions.

## State

Location: `~/.config/pr-watcher/state.json`

```json
{
  "dell/bpa#432": {
    "head_sha": "a1b2c3d",
    "status": "awaiting_approval",
    "worktree": "/home/adam/pr-reviews/dell-bpa-432-a1b2c3d",
    "draft_path": "/home/adam/pr-reviews/dell-bpa-432-a1b2c3d/review-draft.md",
    "first_seen": "2026-05-11T14:23:00Z",
    "review_completed": "2026-05-11T14:25:30Z"
  }
}
```

Dedupe key: `(repo, pr_number, head_sha)`. If a PR's head_sha changes, treat it as a new review event.

Statuses: `seen`, `reviewing`, `awaiting_approval`, `submitted`, `rejected`, `failed`.

Use atomic writes (write to `state.json.tmp`, then `rename`) to avoid corruption on crash mid-write.

## Daemon loop

```
while True:
    for repo in config.repos:
        prs = gh_list_open_prs(repo)
        for pr in prs:
            if pr.author == config.github_user: continue
            if pr.is_draft: continue
            key = (repo, pr.number, pr.head_sha)
            if key in state and state[key].status in ("submitted", "rejected"): continue
            if key in state and state[key].status == "awaiting_approval": continue

            process_pr(repo, pr)

    sleep(config.poll_interval)
```

`process_pr` flow:

1. Mark state as `reviewing`
2. Play `sound_new_pr`, send toast: "New PR: `<repo>#<num>` â€” `<title>`"
3. `git worktree add <worktree_base>/<repo-slug>-<num>-<short-sha> <pr.head_ref>` (clone the PR branch). If repo isn't cloned locally yet, fall back to `gh pr checkout` in a temp directory.
4. Run review pipeline (see below). Write output to `review-draft.md` in the worktree.
5. Mark state as `awaiting_approval`
6. Play `sound_review_ready`, send toast: "Review ready: `<repo>#<num>` â€” run `pr-watcher show <num>`"

If any step fails, mark state as `failed`, log the error to `~/.config/pr-watcher/log.txt`, and continue to the next PR.

## Review pipeline

The review is generated by shelling out to `claude -p`:

```
claude -p --model <config.claude_model> "$(cat review-prompt.md)

PR metadata:
$(gh pr view <num> --repo <repo>)

Diff:
$(gh pr diff <num> --repo <repo>)

Repo conventions:
$(cat <worktree>/CLAUDE.md 2>/dev/null || echo "none")
"
```

Capture stdout. Write to `<worktree>/review-draft.md`.

Default review prompt template (written to `~/.config/pr-watcher/review-prompt.md` on first run):

```
You are reviewing a pull request. Output a markdown document with this structure:

## Verdict
One of: APPROVE | COMMENT | REQUEST_CHANGES

## Summary
2-3 sentences on what this PR does.

## Findings
Bulleted list. Each finding: severity (blocker/concern/nit), file:line, description.
Skip anything that's just style preference. Focus on:
- Correctness bugs
- Edge cases not handled
- Missing or weak tests
- Anything that would page someone at 3am
- Security issues (auth, injection, secrets, etc.)

## Questions for the author
Anything ambiguous that needs clarification before merging.

Be direct. No praise sandwich. No ceremony.
```

User can edit `review-prompt.md` to customize.

## Approval flow

`pr-watcher approve <pr>`:

1. Read `review-draft.md` from the worktree
2. Extract the body (everything after `## Summary`)
3. Run `gh pr review <num> --repo <repo> --approve --body-file <draft>`
4. Update state to `submitted`
5. `git worktree remove <worktree> --force`
6. Print "Submitted."

`pr-watcher comment` and `pr-watcher request-changes` follow the same flow with different `gh pr review` flags.

`pr-watcher reject <pr>`:

1. Update state to `rejected`
2. `git worktree remove <worktree> --force`
3. Print "Discarded."

The user is expected to edit `review-draft.md` before approval if they want to. The Verdict line at the top of the draft is informational only â€” the user picks the actual submission flag via subcommand.

## Audio and notifications (WSL specifics)

**Do not use `notify-send`, `aplay`, `paplay`, `libnotify`, or any Linux audio/notification stack.** WSL2 has no audio session by default.

**Sound:**
```python
subprocess.run([
    "pwsh.exe", "-NoProfile", "-Command",
    f"(New-Object Media.SoundPlayer '{sound_path}').PlaySync()"
], check=False)
```

**Toast notification:**
```python
subprocess.run([
    "pwsh.exe", "-NoProfile", "-Command",
    f"New-BurntToastNotification -Text '{title}', '{body}'"
], check=False)
```

Notification and sound failures must not crash the daemon. Log and continue.

## Daemon mechanics

`pr-watcher start` runs in the foreground by default. The user is expected to wrap it in their own process manager (tmux, nohup, or a systemd user unit they write themselves).

Write a PID file to `~/.config/pr-watcher/pid` on startup. Refuse to start if PID file exists and the process is alive. Remove PID file on graceful shutdown (SIGTERM, SIGINT).

Log to `~/.config/pr-watcher/log.txt`. Rotate at 10MB (keep last 3).

## Dependencies

- `click` (CLI)
- stdlib only otherwise: `subprocess`, `json`, `tomllib`, `pathlib`, `logging`, `time`, `signal`, `datetime`

Do **not** add: `httpx`, `requests`, `PyGithub`, `aiohttp`, `pydantic`, `rich`. All GitHub interaction is via `gh` subprocess.

## Acceptance criteria

The build is done when:

1. `uv run pr-watcher.py --help` lists all subcommands
2. `uv run pr-watcher.py start --once` polls once and exits cleanly with no PRs to process (given valid config and no matching PRs)
3. `uv run pr-watcher.py config` prints resolved config as TOML
4. `uv run pr-watcher.py list` returns valid JSON (empty list is fine)
5. Unit tests cover: config loading, state read/write atomicity, the PR filter logic (skip own PRs, skip drafts, skip already-reviewed), and the `gh` subprocess wrapper (mocked)
6. `pwsh.exe` calls are mockable via a single `notify_sound()` and `notify_toast()` function so tests can run on Linux CI

## Non-goals

- No webhooks. Polling only.
- No web UI.
- No multi-user support.
- No SQLite or any DB.
- No systemd unit (user writes their own).
- No configurable review prompt path beyond what's in config.
- No plugin system.
- No review of the user's own PRs.
- No automatic submission. Human approval gate is mandatory.
- No support for GitHub Enterprise Server (only github.com via `gh` default).
- No retry-with-backoff logic for `gh` failures beyond logging and skipping.
- No support for repos requiring signed commits in the worktree (user handles their own git config).
